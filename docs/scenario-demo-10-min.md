# Scenario demo 10 minutes

## 0-1 min - Cadrage

Afficher les deux URLs Lot 1 et Lot 2. Dire clairement : demo locale, donnees non contractuelles, DAOO prioritaire, points non valides marques `A VALIDER PAOSITRA`.

## 1-3 min - Lot 1 Tresorerie

Se connecter avec le compte Lot 1 regenere. Montrer placements, simulation, creances, budget ou comptes courants selon disponibilite. Insister sur l'audit, les statuts et les exports disponibles.

## 3-5 min - Lot 2 Operations

Se connecter avec le compte Lot 2 regenere. Montrer referentiel agences/postes, caisses, verification, demandes de valeurs et alertes si les donnees demo sont chargees.

## 5-7 min - RBAC

Se connecter avec Consultation Direction ou Audit. Montrer que les actions sensibles sont absentes ou refusees cote API.

## 7-9 min - Audit et tracabilite

Afficher la page Audit. Montrer date, utilisateur, action, objet et resultat disponible. Expliquer que la table est protegee contre update/delete.

## 9-10 min - Clarifications

Afficher `Points a clarifier PAOSITRA`. Conclure sur les donnees attendues : agences/codiques, roles, workflows, regles comptables, formats G59/G60, CPS, CCP/banques, SSO/2FA et criteres de recette.
